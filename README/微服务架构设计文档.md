# 图书管理系统微服务架构设计文档

## 一、项目概述

### 1.1 背景
将现有的单体图书管理系统重构为基于 Spring Cloud 的微服务架构，以提高系统的可扩展性、可维护性和高可用性。

### 1.2 目标
- 实现业务服务的合理拆分
- 保持对外 API 接口与原单体项目完全一致
- 建立完善的认证授权体系
- 实现服务间安全通信

## 二、服务拆分方案

### 2.1 业务领域分析

基于现有单体项目的 Controller 分析，系统包含以下业务领域：

| 业务领域 | 功能描述 | 对应Controller |
|---------|---------|---------------|
| 用户管理 | 用户注册、登录、信息管理等 | UsersController, AuthController |
| 图书管理 | 图书的增删改查、搜索等 | BookController |
| 借阅管理 | 借阅记录的管理、归还等 | BorrowRecordController |
| 评论管理 | 图书评论的发布、查询 | CommentController |
| 货架管理 | 图书分类、货架管理等 | ShelvesController |
| 留言管理 | 用户留言、建议管理 | SuggestionController |
| 人脸识别 | 人脸注册、识别认证 | FaceAuthController |

### 2.3 服务职责定义

#### 2.3.1 auth-service (认证授权服务)
- **端口**: 9001
- **职责**:
  - 用户登录认证
  - JWT令牌生成与验证
  - 令牌刷新
  - 用户注册（基础信息）
- **数据库**: auth_db

#### 2.3.2 user-service (用户管理服务)
- **端口**: 9002
- **职责**:
  - 用户信息管理（CRUD）
  - 用户权限查询
  - 用户人脸数据管理
- **数据库**: user_db

#### 2.3.3 book-service (图书管理服务)
- **端口**: 9003
- **职责**:
  - 图书信息管理（CRUD）
  - 图书搜索
  - 图书库存管理
- **数据库**: book_db

#### 2.3.4 borrow-service (借阅管理服务)
- **端口**: 9004
- **职责**:
  - 借阅记录管理
  - 归还处理
  - 借阅状态查询
- **数据库**: borrow_db
- **依赖**: book-service (查询图书信息), user-service (查询用户信息)

#### 2.3.5 comment-service (评论管理服务)
- **端口**: 9005
- **职责**:
  - 图书评论管理
  - 评论查询
- **数据库**: comment_db
- **依赖**: book-service (验证图书存在), user-service (获取用户信息)

#### 2.3.6 shelves-service (货架管理服务)
- **端口**: 9006
- **职责**:
  - 货架信息管理
  - 图书分类管理
  - 货架与图书关联
- **数据库**: shelves_db
- **依赖**: book-service (查询图书信息)

#### 2.3.7 suggestion-service (留言管理服务)
- **端口**: 9007
- **职责**:
  - 用户留言管理
  - 留言查询
- **数据库**: suggestion_db
- **依赖**: user-service (获取用户信息)

#### 2.3.8 face-service (人脸识别服务)
- **端口**: 9008
- **职责**:
  - 人脸特征提取
  - 人脸比对认证
  - 人脸数据管理
- **数据库**: face_db
- **依赖**: user-service (关联用户信息)


## 五、认证授权体系设计

### 5.1 JWT 令牌结构

```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "username",
    "userId": 123,
    "role": 0,
    "iat": 1704067200,
    "exp": 1704153600,
    "jti": "unique-token-id"
  },
  "signature": "..."
}
```

### 5.2 认证流程

```
┌─────────┐     登录请求      ┌─────────────┐
│  客户端  │ ───────────────> │  Gateway    │
└─────────┘                  └──────┬──────┘
                                    │
                                    ▼
                            ┌───────────────┐
                            │ auth-service  │
                            │  - 验证用户名密码 │
                            │  - 生成JWT令牌   │
                            └───────┬───────┘
                                    │
                                    ▼
                            ┌───────────────┐
                            │  返回JWT令牌   │
                            └───────┬───────┘
                                    │
┌─────────┐     令牌+请求      ┌────▼──────┐
│  客户端  │ ───────────────> │  Gateway  │
│ (存储JWT)│                  │ - 验证JWT  │
└─────────┘                  │ - 路由转发  │
                             └─────┬─────┘
                                   │
                                   ▼
                            ┌─────────────┐
                            │ 业务微服务   │
                            │ - 处理请求   │
                            └─────────────┘
```

### 5.3 服务间调用认证

使用 OpenFeign 拦截器传递用户信息：

```java
@Component
public class FeignRequestInterceptor implements RequestInterceptor {
    
    @Override
    public void apply(RequestTemplate template) {
        // 从当前请求上下文获取JWT
        ServletRequestAttributes attributes = 
            (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        if (attributes != null) {
            HttpServletRequest request = attributes.getRequest();
            String token = request.getHeader("Authorization");
            if (token != null) {
                template.header("Authorization", token);
            }
        }
        
        // 传递用户上下文信息
        String userId = UserContext.getCurrentUserId();
        if (userId != null) {
            template.header("X-User-Id", userId);
            template.header("X-User-Name", UserContext.getCurrentUserName());
            template.header("X-User-Role", UserContext.getCurrentUserRole());
        }
    }
}
```

## 六、数据库设计

### 6.1 数据库拆分

每个微服务拥有独立的数据库，实现数据隔离：

| 服务 | 数据库 | 主要表 |
|------|--------|--------|
| auth-service | auth_db | auth_user, auth_token |
| user-service | user_db | user_info, user_face |
| book-service | book_db | book_info |
| borrow-service | borrow_db | borrow_record |
| comment-service | comment_db | comment_info |
| shelves-service | shelves_db | shelves_info, shelves_book |
| suggestion-service | suggestion_db | suggestion_info |
| face-service | face_db | face_features |

### 6.2 数据一致性

对于跨服务的数据一致性，采用最终一致性策略：
- 使用异步消息队列处理数据同步
- 关键业务采用 Saga 模式处理分布式事务

## 九、安全策略

### 9.1 密码安全
- 使用 BCrypt 加密存储用户密码
- 密码复杂度校验

### 9.2 JWT 安全
- 使用 HS256 算法签名
- 设置合理的过期时间(默认2小时)
- 支持令牌刷新机制
- 使用 HTTPS 传输

### 9.3 接口安全
- 网关层统一认证
- 敏感接口权限控制
- 接口限流防刷
