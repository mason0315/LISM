# 图书管理系统微服务架构

## 项目简介

本项目是一个基于Spring Cloud的图书管理系统微服务架构，使用Spring Boot 3.3.9和Spring Cloud 2023.0.5实现。系统包含多个微服务，每个服务负责不同的业务功能，通过Nacos实现服务发现和配置管理，通过Spring Cloud Gateway实现API网关，通过Sentinel实现断路器，通过Sleuth + Zipkin实现分布式追踪。

## 技术栈

- **Spring Boot**: 3.3.9
- **Spring Cloud**: 2023.0.5
- **Spring Cloud Alibaba**: 2023.0.1.0
- **Nacos**: 服务发现和配置管理
- **Spring Cloud Gateway**: API网关
- **Sentinel**: 断路器
- **MyBatis Plus**: ORM框架
- **MySQL**: 数据库
- **Docker**: 容器化部署

# 下载并启动Nacos

```bash

# 编译项目

cd ../auth-service
mvn spring-boot:run

# 启动其他服务类似...
```

## 容器化部署

### 1. 构建镜像

```bash
# 进入项目根目录
cd microservices

# 编译项目
mvn clean package -DskipTests

# 构建Docker镜像
docker-compose build
```

### 2. 启动服务

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps
```

### 3. 停止服务

```bash
# 停止所有服务
docker-compose down
```

## 注意事项

1. 确保Nacos服务正常运行
2. 确保数据库连接正常
3. 确保所有服务的配置文件正确
4. 启动服务时按照依赖顺序启动
5. 容器化部署时确保Docker环境正常

## 故障排查

1. **服务启动失败**：检查Nacos连接、数据库连接、端口是否被占用
2. **API调用失败**：检查服务是否启动、网关路由是否正确、认证Token是否有效
3. **数据库连接失败**：检查数据库是否启动、连接配置是否正确
4. **Docker容器启动失败**：检查Docker环境、镜像构建是否成功
