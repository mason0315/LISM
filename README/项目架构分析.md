# LIMS 项目架构分析文档

## 一、技术栈分析

### 1.1 后端技术栈（Java + Spring Boot）
- **核心框架**: Spring Boot 3.4.3
- **编程语言**: Java 17
- **持久层框架**: MyBatis 3.0.4
- **数据库**: MySQL 8.0.33
- **构建工具**: Maven
- **API文档**: Knife4j 4.4.0 (基于OpenAPI 3)
- **分页插件**: PageHelper 6.1.1
- **工具库**: Lombok（简化代码）
- **文档处理**: Apache PDFBox 3.0.1, Apache POI 5.2.3
- **JSON处理**: FastJSON 2.0.44
- **AI集成**: 百度AI API（人脸识别、文档处理）

### 1.2 后端技术栈（Python Flask）
- **Web框架**: Flask 2.3.2
- **跨域支持**: Flask-CORS 4.0.0
- **HTTP客户端**: Requests 2.31.0
- **环境配置**: python-dotenv 1.0.0
- **知识图谱**: Neo4j（图数据库）
- **AI增强**: AnythingLLM（大语言模型集成）

### 1.3 前端技术栈（Vue 3）
- **核心框架**: Vue 3.5.17
- **构建工具**: Vite 7.0.4
- **路由管理**: Vue Router 4.0.13
- **状态管理**: Pinia 3.0.3
- **UI组件库**: Element Plus 2.10.4
- **图标库**: Ant Design Icons Vue 7.0.1, FontAwesome 6.7.2
- **HTTP客户端**: Axios 1.10.0
- **人脸识别**: face-api.js 0.22.2
- **知识图谱可视化**: vis-network 9.1.9
- **日期处理**: date-fns 4.1.0
- **开发语言**: TypeScript 5.8.0

## 三、启动入口定位

### 3.1 Java后端启动入口
**文件位置**: `Back/bookmanagement/src/main/java/com/upc/bookmanagement/BookmanagementApplication.java`

**启动流程**:
1. **主类**: `BookmanagementApplication` 类，使用 `@SpringBootApplication` 注解
2. **初始化方法**: `main()` 方法调用 `SpringApplication.run()`
3. **核心配置加载**:
   - 从 `application.yml` 加载数据库配置（MySQL连接信息）
   - 自动扫描 `com.upc.bookmanagement` 包下的所有组件
   - 初始化MyBatis、PageHelper等中间件
   - 加载百度AI配置（API密钥等）
   - 配置Swagger/Knife4j API文档

**关键配置** (`application.yml`):
- 服务端口: 8080
- 数据库: MySQL (localhost:3306/booksmanagement)
- API文档路径: /swagger-ui.html

### 3.2 Python后端启动入口
**文件位置**: `Back(python)/app.py`

**启动流程**:
1. **Flask应用创建**: 使用 `Flask(__name__)` 创建应用实例
2. **CORS配置**: 支持跨域请求（前端端口8080/5173）
3. **Neo4j初始化**: 从环境变量读取连接信息并初始化知识图谱处理器
4. **日志配置**: 配置文件日志和控制台日志
5. **AnythingLLM集成**: 加载大语言模型配置

**关键配置** (`.env`):
- Neo4j连接: URI, 用户名, 密码
- AnythingLLM: API密钥、工作空间ID

### 3.3 Vue前端启动入口
**文件位置**: `Front/vue-bookman/src/main.ts`

**启动流程**:
1. **Vue应用创建**: 使用 `createApp()` 创建Vue实例
2. **插件注册**: 注册路由、Pinia状态管理、Element Plus等
3. **根组件挂载**: 挂载到 `#app` DOM节点

**开发命令**: `npm run dev` (使用Vite启动开发服务器)

## 四、核心调用链路分析

### 4.1 典型业务流程：图书查询

#### 前端发起请求
```
用户操作 → Vue组件 → API封装 → Axios请求
```

**示例**: 用户搜索图书
1. 用户在搜索框输入关键词
2. `SearchList.vue` 组件调用 `api/book.ts` 中的 `searchBooks()` 方法
3. Axios发送GET请求到 `http://localhost:8080/book/search?keyword=xxx&pageNum=1&pageSize=10`

#### 后端处理流程
```
HTTP请求 → Controller → Service → Mapper → 数据库
```

**详细链路**:

1. **Controller层** ([BookController.java](file:///d:\一些东西\LISM\LIMS\LIMS\Back\bookmanagement\src\main\java\com\upc\bookmanagement\controller\BookController.java#L68-L71))
   - 接收HTTP GET请求: `/book/search`
   - 提取参数: keyword（关键词）、pageNum（页码）、pageSize（每页数量）
   - 调用Service层方法: `bookService.searchBooksByKeyword(keyword, pageNum, pageSize)`
   - 封装响应: 使用 `Result.success()` 包装分页结果

2. **Service层** ([BookService.java](file:///d:\一些东西\LISM\LIMS\LIMS\Back\bookmanagement\src\main\java\com\upc\bookmanagement\service\BookService.java#L38-L39))
   - 定义业务接口: `searchBooksByKeyword()`
   - 实现类 ([BookServiceImpl.java](file:///d:\一些东西\LISM\LIMS\LIMS\Back\bookmanagement\src\main\java\com\upc\bookmanagement\service\impl\BookServiceImpl.java))
   - 业务逻辑处理:
     - 使用PageHelper插件自动分页
     - 调用Mapper层查询数据库
     - 返回 `PageInfo<Book>` 分页对象

3. **Mapper层** ([BookMapper.java](file:///d:\一些东西\LISM\LIMS\LIMS\Back\bookmanagement\src\main\java\com\upc\bookmanagement\mapper\BookMapper.java#L38-L39))
   - 定义数据访问接口: `searchBySubstrings()`
   - MyBatis映射文件 ([BookMapper.xml](file:///d:\一些东西\LISM\LIMS\LIMS\Back\bookmanagement\src\main\resources\mapper\BookMapper.xml))
   - SQL执行: 使用模糊查询（LIKE）匹配图书标题、作者、简介等字段

4. **数据库交互**
   - 连接池: Spring Boot自动配置HikariCP
   - SQL执行: `SELECT * FROM book WHERE title LIKE %keyword% OR author LIKE %keyword% ...`
   - 结果映射: 将ResultSet映射为Book实体对象

#### 数据返回流程
```
数据库结果 → Mapper → Service → Controller → JSON响应 → 前端渲染
```

1. **数据库返回**: 查询结果集
2. **Mapper映射**: 转换为 `List<Book>` 对象
3. **Service封装**: 使用 `PageHelper` 包装为 `PageInfo<Book>`
4. **Controller响应**: 转换为JSON格式的 `Result` 对象
5. **前端接收**: Axios接收响应，解析JSON数据
6. **页面渲染**: Vue组件更新DOM，显示图书列表

### 4.2 AI增强查询流程

#### 知识图谱查询链路
```
用户提问 → Python Flask → Neo4j Handler → Neo4j数据库 → 结果格式化 → 返回前端
```

**详细流程**:

1. **前端发起**: 用户在AI问答页面输入问题
2. **Python处理** ([app.py](file:///d:\一些东西\LISM\LIMS\LIMS\Back(python)\app.py)):
   - 接收POST请求到 `/api/knowledge-query`
   - 调用 `generic_neo4j_query(question, depth)` 函数
3. **关键词提取**: 从问题中提取实体、书名、关键词
4. **Neo4j查询** ([neo4j_handler.py](file:///d:\一些东西\LISM\LIMS\LIMS\Back(python)\neo4j_handler.py)):
   - 构建Cypher查询语句
   - 执行图数据库查询
   - 获取实体和关系数据
5. **结果格式化**: 调用 `format_neo4j_results()` 格式化输出
6. **返回前端**: JSON格式返回知识图谱信息

#### 大语言模型增强链路
```
用户问题 → Python Flask → AnythingLLM API → AI模型生成回答 → 返回结果
```

**详细流程**:

1. **用户提问**: 前端发送问题到Python后端
2. **知识检索**: 先查询Neo4j知识图谱获取相关背景信息
3. **LLM调用**: 将问题和知识图谱信息发送到AnythingLLM
4. **AI生成**: 大语言模型生成智能回答
5. **结果返回**: 结合知识图谱和AI回答返回给前端

### 4.3 人脸识别认证流程

```
人脸图像采集 → face-api.js检测特征 → 前端特征提取 → 后端比对 → 认证结果
```

**详细流程**:

1. **前端采集**: [FaceLogin.vue](file:///d:\一些东西\LISM\LIMS\LIMS\Front\vue-bookman\src\views\FaceLogin.vue) 调用摄像头
2. **特征提取**: 使用 [face-api.js](file:///d:\一些东西\LISM\LIMS\LIMS\Front\vue-bookman\src\utils\face.ts) 提取人脸特征向量
3. **后端验证**: [FaceAuthController.java](file:///d:\一些东西\LISM\LIMS\LIMS\Back\bookmanagement\src\main\java\com\upc\bookmanagement\controller\FaceAuthController.java) 接收特征数据
4. **百度AI比对**: 调用百度人脸识别API进行1:N比对
5. **认证结果**: 返回认证成功/失败及用户信息

## 五、架构设计亮点

### 5.1 分层架构设计
- **清晰的职责分离**: Controller（接口层）、Service（业务层）、Mapper（数据层）各司其职
- **高内聚低耦合**: 每层只关注自己的职责，通过接口进行交互
- **易于维护和扩展**: 新增功能只需在对应层添加代码

### 5.2 微服务架构
- **前后端分离**: Vue前端独立部署，Java后端提供RESTful API
- **多语言协作**: Java处理核心业务，Python处理AI和知识图谱
- **服务解耦**: 各服务独立部署，互不影响

### 5.3 AI增强集成
- **知识图谱**: 使用Neo4j存储实体关系，支持复杂关联查询
- **大语言模型**: 集成AnythingLLM提供智能问答
- **人脸识别**: 百度AI API提供生物识别认证

### 5.4 现代化前端技术
- **组件化开发**: Vue组件复用性强
- **响应式设计**: Pinia状态管理，数据驱动视图
- **类型安全**: TypeScript提供编译时类型检查
